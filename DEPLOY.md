# 🚀 Деплой на Railway

## Подготовка к деплою

### 1. Создайте аккаунт на Railway
- Перейдите на [railway.app](https://railway.app)
- Зарегистрируйтесь через GitHub, GitLab или email

### 2. Создайте новый проект
1. Нажмите "New Project" в dashboard
2. Выберите "Deploy from GitHub repo"
3. Подключите ваш репозиторий с мессенджером

### 3. Настройте переменные окружения

В настройках проекта перейдите в "Variables" и добавьте:

```env
DATABASE_URL=postgresql://postgres:password@host:port/railway
JWT_SECRET=your_super_secret_jwt_key_here_make_it_long_and_random
ENCRYPTION_KEY=your_32_character_encryption_key_exactly
CLIENT_URL=https://your-app-name.railway.app
PORT=3000
NODE_ENV=production
MAX_FILE_SIZE=52428800
```

**Важно:**
- `DATABASE_URL` будет автоматически создана при добавлении PostgreSQL сервиса
- `JWT_SECRET` должен быть длинным и случайным (минимум 32 символа)
- `ENCRYPTION_KEY` должен быть ровно 32 символа

### 4. Добавьте PostgreSQL базу данных

1. В проекте Railway нажмите "New Service"
2. Выберите "Add PostgreSQL"
3. Railway автоматически создаст базу данных и добавит `DATABASE_URL` в переменные окружения

### 5. Инициализация базы данных

Railway автоматически запустит сервер, но базу данных нужно инициализировать. Добавьте в `server.js`:

```javascript
// В начале файла, после require
const { initDatabase } = require('./models/database');

// Перед запуском сервера
initDatabase().then(() => {
  const PORT = process.env.PORT || 3000;
  server.listen(PORT, () => {
    console.log(`Server running on port ${PORT}`);
  });
});
```

## Деплой

### Автоматический деплой

После настройки переменных окружения Railway автоматически:
1. Соберет проект с помощью Nixpacks
2. Установит зависимости
3. Запустит сервер

### Первый деплой может занять 5-10 минут

## Настройка домена

### Бесплатный домен Railway

1. Перейдите в настройки вашего сервиса
2. Во вкладке "Settings" найдите "Domain"
3. Ваш домен будет вида `your-app-name.railway.app`

### Своё доменное имя

1. Нажмите "Custom Domain"
2. Введите ваш домен (например, `messenger.yourdomain.com`)
3. Настройте DNS записи:
   ```
   Type: CNAME
   Name: messenger
   Value: proxy.railway.app
   ```

## SSL сертификат

Railway автоматически предоставляет SSL сертификаты для:
- Бесплатных доменов `.railway.app`
- Пользовательских доменов

## Траблшутинг

### Ошибка: Database connection failed

**Решение:**
1. Проверьте что `DATABASE_URL` правильный
2. Убедитесь что PostgreSQL сервис запущен
3. Проверьте переменные окружения

### Ошибка: JWT_SECRET is required

**Решение:**
1. Добавьте `JWT_SECRET` в переменные окружения
2. Используйте генератор случайных строк
3. Минимум 32 символа

### Медленная загрузка

**Решение:**
1. Проверьте логи на ошибки
2. Убедитесь что все зависимости установлены
3. Рассмотрите upgrading plan для большей мощности

## Безопасность

### HTTPS

Railway автоматически настраивает HTTPS для всех доменов.

### Переменные окружения

Все чувствительные данные хранятся в переменных окружения:
- Никогда не храните секреты в коде
- Используйте сложные пароли
- Регулярно обновляйте ключи

### Rate Limiting

Приложение уже настроено с rate limiting:
- 100 запросов в 15 минут на IP
- 5 попыток аутентификации в 15 минут

## Резервное копирование

### База данных

Railway автоматически создает бэкапы PostgreSQL:
- Ежедневные бэкапы
- Хранятся 30 дней
- Можно скачать через dashboard

### Файлы

Загруженные файлы хранятся в `/uploads`:
- Они не сохраняются между деплоями
- Для продакшена используйте облачное хранилище (S3, Cloudinary)

## Обновление приложения

### Автоматические обновления

При пуше в основную ветку Railway автоматически:
1. Соберет новую версию
2. Проведет health check
3. Переключит трафик на новую версию

### Ручной деплой

1. Пушьте изменения в GitHub
2. В dashboard Railway нажмите "Redeploy"
3. Следите за процессом в логах
