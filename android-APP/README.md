# Flux - Android Messenger Application

## ✅ Production Ready with Railway

**Railway URL:** `https://funi-production.up.railway.app/`

## Описание

Flux - это полнофункциональное Android приложение-мессенджер, перенесённое с веб-версии. Приложение включает поддержку отправки сообщений, звонков, поиска, и реал-тайм синхронизации через Socket.IO.

**Статус:** ✅ **100% готово на Railway (Production)**

## Возможности

### ✅ Реализованные функции

1. **Аутентификация**
   - Регистрация нового пользователя
   - Вход в аккаунт
   - Сохранение сессии
   - Выход из приложения

2. **Сообщения**
   - Просмотр списка чатов
   - Отправка и получение сообщений в реал-тайм
   - Поддержка зашифрованных сообщений (AES)
   - Статусы сообщений (отправлено, доставлено, прочитано)
   - Реакции на сообщения (эмодзи)
   - Ответы на сообщения (цитирование)

3. **Звонки**
   - Инициирование звонков (голосовые и видео)
   - Ответ на входящие звонки
   - Отклонение звонков
   - Завершение звонков
   - Основана на WebRTC

4. **Поиск**
   - Поиск сообщений
   - Поиск чатов
   - Поиск пользователей
   - Фильтрация результатов в реал-тайм

5. **Реал-тайм обновления**
   - Socket.IO для мгновенной синхронизации
   - Уведомления о статусе пользователя (онлайн/офлайн)
   - Индикаторы печати
   - Доставка и прочтение сообщений

6. **Дополнительное**
   - Поддержка папок
   - Загрузка медиа файлов
   - Профиль пользователя
   - Безопасное хранение токенов

## Требования

- Android SDK 34 или выше
- Java 8 или выше
- Android Studio 2024 или выше

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/yourusername/flux.git
cd flux/android-APP
```

### 2. Настройка Backend URL

Измените базовый URL в файле `RetrofitClient.java`:

```java
private static final String BASE_URL = "https://your-backend-url:3000/";
```

Для локального тестирования используйте IP-адрес вашей машины вместо localhost.

### 3. Сборка и запуск

```bash
./gradlew build
./gradlew installDebug
```

Или запустите прямо из Android Studio (Run > Run 'app').

## Конфигурация

### Переменные окружения

Создайте файл `local.properties` в корне проекта:

```properties
sdk.dir=/path/to/android/sdk
```

### Backend конфигурация

Убедитесь, что backend работает и доступен по адресу, указанному в `RetrofitClient.java`.

## Использование приложения

### Регистрация

1. Запустите приложение
2. Перейдите на экран "Sign Up"
3. Заполните все требуемые поля
4. Нажмите "Register"

### Вход

1. На экране входа введите email и пароль
2. Нажмите "Login"
3. После успешного входа вы будете перенаправлены на главный экран

### Отправка сообщений

1. Выберите чат из списка
2. Введите сообщение в поле ввода
3. Нажмите "Send"

### Звонки

1. Откройте чат с нужным пользователем
2. Нажмите кнопку "Call" (голосовой звонок) или "Video" (видеозвонок)
3. Другой пользователь получит входящий звонок
4. Нажмите "Answer" для принятия или "Reject" для отклонения

### Поиск

1. Перейдите на экран поиска
2. Введите поисковый запрос
3. Результаты будут отображаться в реал-тайм

## API endpoints

### Аутентификация
- `POST /api/register` - Регистрация нового пользователя
- `POST /api/login` - Вход в аккаунт
- `POST /api/auth/logout` - Выход из приложения

### Пользователи
- `GET /api/users` - Получить список всех пользователей
- `GET /api/users/{id}` - Получить информацию пользователя
- `PUT /api/users/{id}` - Обновить профиль пользователя
- `POST /api/users/search?q=query` - Поиск пользователей

### Чаты
- `GET /api/chats` - Получить список чатов
- `POST /api/chats` - Создать новый чат
- `GET /api/chats/{id}` - Получить информацию о чате
- `PUT /api/chats/{id}` - Обновить чат
- `DELETE /api/chats/{id}` - Удалить чат

### Сообщения
- `GET /api/messages?chatId={id}` - Получить сообщения чата
- `POST /api/messages` - Отправить сообщение
- `DELETE /api/messages/{id}` - Удалить сообщение
- `POST /api/messages/{id}/reactions` - Добавить реакцию
- `DELETE /api/messages/{id}/reactions/{reactionId}` - Удалить реакцию

### Звонки
- `POST /api/calls` - Инициировать звонок
- `POST /api/calls/{id}/answer` - Ответить на звонок
- `POST /api/calls/{id}/reject` - Отклонить звонок
- `POST /api/calls/{id}/end` - Завершить звонок

### Файлы
- `POST /api/upload` - Загрузить файл

### Поиск
- `GET /api/search?q=query` - Поиск сообщений, чатов и пользователей

## Socket.IO события

### Клиент отправляет

- `user:online` - Сообщить, что пользователь онлайн
- `message:queue` - Отправить сообщение
- `message:reaction` - Добавить реакцию
- `presence:typing` - Отправить индикатор печати
- `message:delivered` - Сообщить о доставке
- `message:read` - Сообщить о прочтении
- `chat:join` - Присоединиться к чату
- `chat:leave` - Покинуть чат

### Клиент получает

- `new_message` - Новое сообщение
- `message:reaction` - Новая реакция
- `presence:typing` - Пользователь печатает
- `call:incoming` - Входящий звонок
- `users:online` - Список онлайн пользователей
- `message:delivered` - Доставка сообщения
- `message:read` - Прочтение сообщения

## Безопасность

### Шифрование

- Сообщения шифруются с помощью AES-256
- Токены хранятся в SharedPreferences (можно улучшить используя EncryptedSharedPreferences)
- HTTPS для всех запросов

### Лучшие практики

1. Всегда используйте HTTPS в продакшене
2. Регулярно обновляйте зависимости
3. Используйте ProGuard/R8 для обфускации
4. Проверяйте все входные данные

## Развертывание

### Railway

По умолчанию приложение подключается к backend, развёрнутому на Railway. Убедитесь, что:

1. Backend запущен и доступен
2. URL backend правильно сконфигурирован
3. CORS правильно настроен на backend

## Решение проблем

### Приложение не подключается к backend

1. Проверьте URL в `RetrofitClient.java`
2. Проверьте соединение с интернетом
3. Убедитесь, что backend работает
4. Проверьте логи в Logcat

### Сообщения не отправляются

1. Проверьте интернет соединение
2. Убедитесь, что вы авторизованы
3. Проверьте Socket.IO соединение
4. Проверьте токен доступа

### Звонки не работают

1. Убедитесь, что у приложения есть разрешение на доступ к микрофону/камере
2. Проверьте наличие WebRTC поддержки
3. Убедитесь, что оба пользователя онлайн
